    	GIT  COMMANDS..

1. git --version/git -v --> (this command is to know the version of the installed git bash)
repositories--> this are just like simple folder(VIRTUAL TEMPORARY STORAGE) for our projects in git to track all the modifications and updates for our project..

2. clone --> to copy the repo/file from GitHub remote to local(GitHub desktop)
	command: git clone <url of repo>

3. status -> command: get status
	-untracked
	-modified
	-unmodified
	-staged
4. Add
   	command : git add <filename/directories>
	ex: git add \.index.html
	Add is in between 
	untracked ->|Add| ->commit

5. Commit
	-commit is used to finalize the changes
	-command: git commit -m "<your commit msg>"

6. Push
	-command-> git push origin main

7. Pull
	-git pull <remote_name> <branch_name>:  Fetches changes from a remote repository and merges them into the current branch. 

8. git show <commit_id>: Displays the details of a specific commit. 

9. git blame <file_name>: Shows who last modified each line of a file.

Resloving Merge conflicts:

An Event that takes place when git is unable to automatically resolve differences in code between two commits.

Undoing changes:
~~~~~~~~~~~
case 1: staged changes
	git reset <-file name->
	git reset
case 2: Commited changes(for one commit)
	git reset Head~1
case 3: Commited changes(for many commits)
	git reset<- commit hash->
	git reset --hard<-commit hash->

git log (check previous commits)

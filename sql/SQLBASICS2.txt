KEYS:
~~~~~~~~~~~~ 
1. Primary key : it is a column in a table that uniquely identifies each row.(an unique id).
-- only one pk and that should not be null

2. foreign key: A foreign ley is a column or set of columns in a table that refers to the primary key of an other table.
-- there can be multiple fks.
-- foreign can have duplicate and null values.

	CONSTRAINTS:
	~~~~~~~~~~~~~
SQL CONSTRAINTS ARE USED TO SPECIFY THE RULES OF DATA IN A TABLE.

*NOT NULL : COLUMN CANNOT HAVE A NULL VALUE
*UNIQUE : ALL VALUES MUST BE DIFFERENT 
*PRIMARY KEY: UNIQUE AND NOT NULL.
*FOREIGN KEY : PREVENT ACTIONS THAT WOULD DESTROY LINKS BETWWEEN TABLES
	EX: CREATE TABLE TEMP(
			CUST_ID INT,
		FOREIGN KEY(CUST-ID) REFERENCES CUSTOMER(ID));
	FROM THE ABOVE EXAMPLE WE CAN SEE THAT THERE IS AN COUSTMER TABLE WITH ID AS PRIMARY KEY SO THAT THE CUST_ID WILL BE FOREIGN 
KEY FOR THE TABLE TEMP...WHICH REFERENCE FROM COUSTOMER TABLE..

*DEFAULT : WE CAN ASSIGN DEFAULT VALUES AS WELL...
EX COLUMN NAME DATA TYPE  DEFAULT VALUE(BASED ON THE DATATYPE)

* CHECK CONSTRAINT: IT CAN LIMIT THE VALUES ALLOWED IN A COLUMN

ex:CREATE TABLE CITY(
id int primary key,
city varchar(50) not null,
age int ,
constraint check_age check(age>=18 and city = "Delhi"));

 if we try to add something like data with age less then 18 or city other then delhi ...check constraint is violated and data is not added to table...

Cascading for fk
~~~~~~~~~
on delete cascade : When we create a foreign key using the option, it deletes the referencing rows in the child table when the refrenced row is deleted in the parent table which has primary key.

on update cascade: when we create a foreign key using update cascade the referencing rows are updated in the child table when the referenced row is updated in the parent table which has a primary key.

Joins in sql:
~~~~~~~
join is used to combine rows from two or more tables, based on a related columns between them.
* inner join:
 		it returns records that have matching values in both tables.
		syn: select columns from table a inner join tableb on tablea.column = table b.column

*left join:
		Returns all records from the left table,and the matched records from the right table.
		syn: Select columns(s) from tableA left join tableB on tableA.col= tableB.col;

* Right join -----outerJoin()
		returns all records from the right table,and the matched records from the left table.
		Syn: select columns(s) from tableA right join tableB on tableA.col=tableB.col
* full join....

Self join:
	It is a regular join but the table is joined with itself..
	syn:  select column from table as a join table as b on a.col.1=b.col.2;

Union:
~~~~~~
It is used to combine the result-set of two or more select statments.
Gives unique records
rules:
* same no of columns for both the tables
*columns must have similar datatypes
*columns in every select should be in same order

Syn:
select columns from tablea union select columns from tableb.
replace with union all to get duplicate ,repeated  values as well
